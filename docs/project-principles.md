# Project Principles

- Open engineering: share CAD, firmware, electronics, artwork, and documentation openly.
- Documentation-first: write and organize docs early to make the build reproducible.
- Modular design: build each subproject as a reusable, serviceable module.
- Validate early: prototype one digit and a small batch of flaps before scaling.
- Mechanical authenticity: prioritise real split-flap movement over screen effects.
- Reuse shared assets: place common artwork, fonts, symbols, and libraries in `shared/`.
- Respect trademark: use Project 108 as the umbrella brand, not direct LOST branding.
- Incremental progress: break work into small, reviewable steps and capture decisions in writing.
